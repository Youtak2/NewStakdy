#FORK
현재 이 저장소를 fork 한다.
생성된 원격 저장소에 이름 혹은 GitHub ID로 폴더를 생성한다.
생성된 폴더에 자신의 소스코드를 업로드 한다. (주의 폴더명에 콜론":" 쓰지 않을 것!)
commit 규칙을 지키도록 한다!
원본 저장소로 Pull Request를 한다.
스터디장의 PR Merge가 끝나면 원본 저장소가 업데이트 되므로, 다음 주차에 PR을 보내기 전 git fetch로 동기화 할 것!
o https://velog.io/@jisubin12/Github-외부저장소-fork-pull-request-동기화-하기
타인의 PR을 보고 자유롭게 코드리뷰를 한다.
주의!! 기존 폴더(혹은 타인의 폴더)를 지우지 않도록 유의! (개인 폴더 내부만 수정하기)
COMMIT 형식
"[문제 출처(플랫폼)] 문제이름 / 문제번호 / 난이도 / 걸린시간"
EX) git commit -m "[SEA] 농작물 수확하기 / 2805 / D3 / 60분"
백준 BOJ /삼성 SWEA-> SWEA /기타 ETC
• 매일 코딩문제 풀이가 끝나면 fork한 저장소에 Push한 후 Pull request를 보낼 것
• PR은 해당 주차 일요일에 스터디 이후 merge 예정
• PR 제목에 해당 주차에 푼 문제 개수를 누적 업데이트 할 것
